# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Описание проекта

Telegram-бот для выдачи лид-магнитов (файлов, ссылок, текстов) пользователям в обмен на подписку на Telegram-каналы. Написан на Python с использованием aiogram 3.x и SQLite.

## Запуск бота

```bash
# Активировать виртуальное окружение
source .venv/bin/activate

# Установить зависимости
pip install -r requirements.txt

# Запустить бота
python bot.py
```

## Переменные окружения (.env)

```
TOKEN_BOT=<токен Telegram-бота от @BotFather>
ADMIN_IDS=<id1,id2,...>  # через запятую, без пробелов
```

## Архитектура

### Точка входа — `bot.py`
Импортирует хэндлеры (регистрация происходит при импорте), запускает polling через `dp.start_polling(bot)`.

### `config.py`
Создаёт и экспортирует глобальные объекты `bot` (экземпляр `Bot`) и `dp` (экземпляр `Dispatcher`), а также список `ADMIN_IDS`. Все модули импортируют их отсюда — не создают собственные экземпляры.

### `database.py`
Синхронный SQLite через стандартный `sqlite3` (не `aiosqlite`). Инициализация таблиц происходит при импорте модуля (`init_db()` в конце файла).

Три таблицы:
- `users` — зарегистрированные пользователи
- `channels` — каналы с лид-магнитами (`magnet_type`: `document` / `link` / `text` / `photo`)
- `rewards_history` — история выдач (UNIQUE по паре `user_id + channel_id`)

### `messages.py`
Все пользовательские тексты хранятся здесь как константы. При изменении текстов редактировать только этот файл.

### `handlers/`

Регистрация хэндлеров происходит при импорте модуля — либо через `@dp.callback_query(...)` декоратор, либо через `dp.message.register(...)` в конце файла.

- **`start.py`** — обработчик `/start`, сборка главного меню каналов. Экспортирует `send_channel_menu()`, которую используют другие хэндлеры для возврата в меню.
- **`callbacks.py`** — пользовательский flow: открыть канал → проверить подписку → получить лид-магнит → повторно скачать. Callback data формата `channel:<action>:<id>`.
- **`admin.py`** — панель администратора (только для `ADMIN_IDS`). Использует FSM (`aiogram.fsm`) для многошаговых диалогов: добавление/редактирование/удаление каналов, рассылка, статистика. Команда входа — `/admin`.

### Поток подписки на канал
1. `/start` → меню с кнопками каналов
2. Нажатие на канал (`channel:open:<id>`) → бот предлагает подписаться и показывает кнопку проверки
3. Нажатие «Я подписался» (`channel:check:<id>`) → бот вызывает `bot.get_chat_member()` для проверки
4. Если подписан — `_send_lead_magnet()` отправляет материал и записывает в `rewards_history`
5. Повторный просмотр через `channel:view_rewards` → `channel:reward:<id>`

## Важные детали

- `chat_identifier` в таблице `channels` — числовой `chat_id` или строка `@username`. Проверку подписки делает `_resolve_chat_identifier()` в `callbacks.py`.
- Лид-магнит типа `document` / `photo` передаётся через `file_id` Telegram (не путь к файлу). Для получения `file_id` есть вспомогательный скрипт `get_file_id.py`.
- Поле `button_title` — отдельное название кнопки в меню; если пустое, используется `title` канала.
- Бот не использует `aiosqlite` — все вызовы к БД синхронные и должны оставаться таковыми.
